# Checklist — Nivel Táctico

> Validación verificable del nivel táctico (RV-01…RV-05).

## Cobertura y Trazabilidad

- [ ] Los 16 casos de uso CU-T01…CU-T16 están especificados.
- [ ] Cada CU-T traza a un OE (OE1–OE4) o al objetivo transversal OT5.
- [ ] Cada CU-T tiene al menos una historia de usuario (HU-T-*).
- [ ] Cada CU-T tiene criterios de aceptación Dado/Cuando/Entonces.

## Calidad de Especificación

- [ ] Requisitos funcionales (RF-T-01..16) verificables.
- [ ] Requisitos no funcionales (RNF-T-*) definidos.
- [ ] Reglas de negocio referenciadas (RN-01/03/04/06/07/10).

## Ecosistema y APIs (OE2)

- [ ] Catálogo de APIs versionado y documentado.
- [ ] Integraciones con aprovisionamiento seguro de credenciales.
- [ ] Planes SaaS del marketplace definidos.

## Cloud y SLA (OE3)

- [ ] Umbrales de SLA validados y monitoreados con alertas.

## Datos e Inteligencia (OE4)

- [ ] DWH con calidad y linaje; KPIs y tableros configurables.

## Seguridad y Cumplimiento (OT5)

- [ ] Roles/permisos institucionales con privilegio mínimo y MFA.
- [ ] Auditoría y cumplimiento gestionables.

## Inmutables y Calidad Visual

- [ ] No se modificó el alcance B2G (RN-03) ni OE1–OE4 (RN-04).
- [ ] Tableros y catálogos legibles y proporcionados (RI-04).
